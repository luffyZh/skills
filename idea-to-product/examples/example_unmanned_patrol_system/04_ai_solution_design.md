# 04 AI Solution Design

## 1. AI 在产品中的角色

- 目标检测与识别
- 异常事件初判
- 任务触发与调度建议
- 会话式事件摘要与告警生成
- 多设备协同辅助

## 2. 任务拆解

| 任务 | 是否适合交给 AI | 原因 | 输出 |
|---|---|---|---|
| 人员 / 异常目标识别 | 是 | 视觉模型适合感知识别 | 目标框、标签、置信度 |
| 事件风险分级 | 是 | 可结合规则与模型输出判断 | 风险级别 |
| 调度建议 | 是 | 可结合位置、距离、任务状态做推荐 | 调度建议 |
| 自动处置决策 | 否 | 高风险动作不应完全自动化 | 人工确认结果 |

## 3. Prompt 策略

### System Prompt 原则

- 对不确定识别结果保持保守
- 所有调度建议必须带理由
- 高风险事件必须建议人工确认

### 关键提示词模块

- 当前场景配置
- 设备状态
- 目标类型与风险等级
- 输出格式和告警结构

## 4. Skills / 工具调用策略

| 能力 | 调用时机 | 目的 | 输出 |
|---|---|---|---|
| 视觉识别模型 | 无人机发现目标时 | 识别疑似目标 | 识别结果 |
| 规则引擎 | 识别之后 | 做风险分级与调度判断 | 风险级别 / 调度条件 |
| 协同调度模块 | 事件成立后 | 调度机器狗复核 | 调度任务 |
| 事件摘要生成 | 复核后 | 生成中控可读摘要 | 告警摘要 |

## 5. Agent / Workflow 设计

### 多 Agent 方案

- `Perception Agent`：处理视频流目标识别
- `Risk Agent`：做事件初判和等级评估
- `Dispatch Agent`：根据规则调度机器狗
- `Review Agent`：生成中控摘要并保留人工确认入口

### 协作顺序

1. `Perception Agent`
2. `Risk Agent`
3. `Dispatch Agent`
4. `Review Agent`

### 设备角色定义

- 无人机：广域巡查、一级发现、目标初始锁定
- 机器狗：地面接近、近距复核、细节回传

### 触发与调度逻辑

- 无人机发现疑似人员或异常目标时触发识别
- 若风险达到阈值，则调度最近机器狗前往
- 机器狗复核后回传视频和状态
- 高风险事件由中控人工确认后升级处置

## 6. 输入输出约束

### 输入

- 视频流
- 设备位置与状态
- 园区地图与禁入区配置

### 输出

- 风险告警
- 调度记录
- 复核结果
- 中控摘要

## 7. 风险与兜底机制

- 夜间识别不稳定时提高人工确认比例
- 机器狗不可达时回退为持续空中跟踪 + 人工派遣
- 高风险动作不做全自动处置
- 保留“人工接管 / 暂停调度 / 取消任务”开关
