# MVP PLAN

## 1. MVP Demo 目标

清晰展示“无人机发现异常 -> 机器狗复核 -> 中控确认”的协同巡航价值。

## 2. 核心演示主线

1. 展示园区地图和重点区域
2. 无人机按路线巡航
3. 识别到疑似入侵目标
4. 系统自动触发机器狗复核
5. 中控接收告警和复核画面
6. 人工确认事件

## 3. 必做功能

- 巡航区域配置
- 无人机任务展示
- 疑似目标识别
- 机器狗调度与回传
- 告警摘要展示

## 4. 可延后功能

- 跨多个区域的复杂策略联动
- 多台机器狗竞争调度
- 自动派工和工单系统联动

## 5. 实现优先级

| 项目 | 目标 | 优先级 | 备注 |
|---|---|---|---|
| 地图与巡航展示 | 先讲清场景 | 高 | MVP 必做 |
| 异常识别 | 体现 AI 能力 | 高 | MVP 必做 |
| 机器狗调度 | 体现协同价值 | 高 | MVP 必做 |
| 中控界面 | 体现上层系统价值 | 高 | MVP 必做 |
| 自动处置 | 非当前重点 | 低 | 延后 |

## 6. 演示建议

- 演示时不要先讲硬件参数，先讲安防问题
- 强调“发现到确认”的时间缩短
- 高亮上层系统的价值，而不是把重点放在本体设备炫技
