# DESIGN

## 1. 系统目标

支撑园区重点区域入侵巡航场景下的核心链路：无人机发现疑似目标 -> AI 初判 -> 机器狗近距复核 -> 中控确认与上报。

## 2. 设计范围

### 本次覆盖

- 园区地图和重点区域建模
- 无人机巡航任务管理
- 目标识别与风险初判
- 机器狗调度复核
- 中控事件回传

### 本次不覆盖

- 多园区统一大平台
- 自动处置执行
- 复杂跨品牌设备编队

### 本体设备边界

- 设备本体负责飞行、行走、视频回传和基础控制
- 上层系统负责识别、事件判断、调度策略和业务闭环

## 3. 总体架构

### 架构说明

系统由设备接入层、感知与事件层、调度编排层和中控应用层组成。

### 核心模块

| 模块 | 职责 | 输入 | 输出 |
|---|---|---|---|
| Device Access | 接入无人机与机器狗状态和视频流 | 设备数据 | 标准化设备状态 |
| Perception Engine | 识别目标与异常行为 | 视频流 | 识别结果 |
| Event Engine | 风险分级与触发规则 | 识别结果、规则 | 事件 |
| Dispatch Engine | 调度机器狗复核 | 事件、设备状态 | 调度任务 |
| Control Center | 呈现告警与结果 | 事件、复核结果 | 中控界面 |

### 设备角色与协同关系

- 无人机：高空广域巡查、一级发现
- 机器狗：地面接近、近距复核

## 4. 数据流 / 信息流

1. 用户配置巡航区域和任务
2. 无人机执行巡航并回传视频流
3. 感知引擎识别疑似目标
4. 事件引擎做风险初判
5. 调度引擎分配机器狗前往复核
6. 中控接收复核结果并触发后续动作

### 事件触发链路

1. 发现异常
2. AI 判定风险级别
3. 调度协同设备
4. 人工确认 / 告警升级 / 处置闭环

## 5. 外部依赖

- 无人机本体与 SDK
- 机器狗本体与 SDK
- 视觉识别模型
- 地图与定位能力

## 6. 技术选型理由

- 设备 SDK 提供本体控制和状态能力
- 视觉模型承担人员和异常目标识别
- 规则引擎用于控制高风险场景下的触发和人工接管

## 7. 风险与约束

- 夜间和复杂环境影响识别质量
- 不同设备品牌的接口适配工作量较大
- 调度稳定性需要长期验证
- 现场部署依赖网络、电池和环境条件
- 人工接管要求不可省略
