# 04 AI Solution Design

## 1. AI 在产品中的角色

- 会议内容理解
- 摘要生成
- 行动项提取
- 会后跟进建议生成

## 2. 任务拆解

| 任务 | 是否适合交给 AI | 原因 | 输出 |
|---|---|---|---|
| 会议摘要 | 是 | 大模型擅长长文本压缩 | 摘要 |
| 行动项提取 | 是 | 适合从上下文中抽取待办 | 待办清单 |
| 责任人建议 | 是 | 可基于发言内容给建议 | 责任人建议 |
| 最终确认 | 否 | 需要人工兜底 | 审核结果 |

## 3. Prompt 策略

### System Prompt 原则

- 优先提取可执行信息
- 输出必须结构化
- 不确定时显式标注“需人工确认”

### 关键提示词模块

- 会议背景
- 输出格式定义
- 行动项抽取规则
- 责任人与截止时间判断规则

## 4. Skills / 工具调用策略

| 能力 | 调用时机 | 目的 | 输出 |
|---|---|---|---|
| 语音转录 | 会议导入后 | 获取文本 | 转录稿 |
| 摘要生成 | 转录完成后 | 获取纪要草稿 | 摘要 |
| 行动项抽取 | 摘要后 | 形成可执行结果 | 行动项列表 |

## 5. Agent / Workflow 设计

### 多 Agent 方案

- `Transcript Agent`：处理会议文本
- `Summary Agent`：输出会议摘要
- `Action Agent`：提取行动项与责任建议
- `Review Agent`：做最终质量检查

### 协作顺序

1. `Transcript Agent`
2. `Summary Agent`
3. `Action Agent`
4. `Review Agent`

## 6. 输入输出约束

### 输入

- 会议音频或文本
- 会议上下文

### 输出

- 摘要
- 行动项列表
- 会后同步文案

## 7. 风险与兜底机制

- 音频质量差时允许用户补充文本
- 责任人和截止时间无法判断时标记待确认
- 在最终发送前提供人工审核步骤
