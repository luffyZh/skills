# DESIGN

## 1. 系统目标

支撑 AI 会议助手 MVP 的核心链路：导入会议内容 -> 生成摘要 -> 提取行动项 -> 输出会后同步结果。

## 2. 设计范围

### 本次覆盖

- 会议内容输入
- 摘要生成
- 行动项抽取
- 审核与导出

### 本次不覆盖

- 企业级账号体系
- 与多个协作平台的深度集成

## 3. 总体架构

### 架构说明

系统由输入层、AI 处理层和输出层组成。

### 核心模块

| 模块 | 职责 | 输入 | 输出 |
|---|---|---|---|
| Input Module | 接收音频或文本 | 会议内容 | 原始文本 |
| Summary Module | 生成会议摘要 | 文本 | 摘要 |
| Action Module | 抽取行动项 | 文本 / 摘要 | 待办列表 |
| Review Module | 人工确认 | 摘要与待办 | 最终结果 |

## 4. 数据流 / 信息流

1. 用户上传会议内容
2. 转录或读取文本
3. 大模型生成摘要
4. 大模型提取行动项
5. 用户审核并导出

## 5. 外部依赖

- 语音转录服务
- 大语言模型

## 6. 技术选型理由

- 语音转录服务负责把会议音频转成文本
- 大语言模型负责理解上下文和生成结构化结果

## 7. 风险与约束

- 音频质量影响结果
- 大模型输出可能不稳定
- 隐私与数据合规需提前考虑
